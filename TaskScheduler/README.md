# 작업 스케줄러 정렬

## 문제

할 일 관리 앱을 구현합니다. `TodoTask` 클래스를 만들어 여러 기준으로 자동 정렬되도록 합니다.

> `System.Threading.Tasks.Task`와 이름 충돌을 방지하기 위해 클래스명을 `TodoTask`로 사용합니다.

## 요구사항

- `TodoTask` 클래스에 `Title` (string), `Priority` (int, 높을수록 긴급), 
`DueDate` (string, "YYYY-MM-DD" 형식), `IsCompleted` (bool) 필드를 포함

- `IComparable<TodoTask>` 인터페이스를 구현하여 3중 기준으로 정렬
  - 1차: Priority 내림차순 (높은 우선순위가 먼저)
  - 2차: DueDate 오름차순 (마감일이 빠른 것이 먼저)
  - 3차: Title 오름차순 (사전순)
	
- `ToString()` 오버라이드: 작업 정보를 보기 좋게 출력

## 예상 실행 결과

```
=== 정렬 전 ===
  [우선순위 3] 보고서 작성 (마감: 2026-03-15)
  [우선순위 1] 이메일 확인 (마감: 2026-03-10)
  [우선순위 3] 버그 수정 (마감: 2026-03-10)
  [우선순위 2] 회의 준비 (마감: 2026-03-12)
  [우선순위 3] 코드 리뷰 (마감: 2026-03-10)

=== 정렬 후 ===
  [우선순위 3] 버그 수정 (마감: 2026-03-10)
  [우선순위 3] 코드 리뷰 (마감: 2026-03-10)
  [우선순위 3] 보고서 작성 (마감: 2026-03-15)
  [우선순위 2] 회의 준비 (마감: 2026-03-12)
  [우선순위 1] 이메일 확인 (마감: 2026-03-10)
```
